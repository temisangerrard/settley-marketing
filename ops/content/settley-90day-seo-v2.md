## 3. Article Production Standards (NEW)

### Template Usage (Pattern v3)

Every article must use `settley-article-template.html` with the **same section order**. This is now a fixed pattern, not freeform writing.

Editorial structure and page-quality rules are defined in:

- `content/templates/mira-article-blueprint.md`

If the shell template and the blueprint conflict, follow the blueprint for hierarchy, pacing, and CTA discipline.

**Core variables to fill first (before drafting):**

```
{{META_TITLE}}         — 55-60 chars, keyword near front
{{META_DESCRIPTION}}   — 150-160 chars, keyword + value
{{ARTICLE_TITLE}}      — H1
{{ARTICLE_SUBTITLE}}   — payoff line
{{ARTICLE_CATEGORY}}   — cluster/category label
{{PUBLISH_DATE}}       — month + year
{{READ_TIME}}          — 200 wpm estimate
{{UPDATED_DATE}}       — same as publish initially
{{CTA_HEADLINE}}       — waitlist value proposition
{{CTA_COPY}}           — one-line CTA support text
{{CTA_LINK}}           — usually /signup
{{CTA_BUTTON_LABEL}}   — e.g. "Join the waitlist →"
```

### Required article pattern (all SEO content)

1. Risk callout (investment pages only)
2. Stat row (optional)
3. Key takeaways (3–5 bullets)
4. Intro section (keyword in first 100 words)
5. Core explanation section
6. Practical application section
7. Risks/constraints section
8. Comparison/data table (optional)
9. Action steps section
10. FAQ section (when query intent is question-heavy)
11. Bottom line section
12. Waitlist CTA strip
13. Sources section (3+ authoritative links)

### Which template components to use per article type

| Article type | Key points box | Stat row | Timeline | Tables | FAQ schema |
|-------------|---------------|---------|----------|--------|-----------|
| Pillar / ultimate guide | ✅ | ✅ | optional | ✅ | optional |
| How-to | ✅ | optional | optional | optional | optional |
| Regulatory/explainer | ✅ | ✅ | ✅ | ✅ | optional |
| FAQ post | ✅ | optional | optional | optional | ✅ |
| Comparison | ✅ | optional | optional | ✅ | optional |
| Risk / trust piece | ✅ | optional | optional | optional | optional |

### Pre-publish checklist

- [ ] Template applied — article renders correctly on mobile
- [ ] Blueprint followed — `content/templates/mira-article-blueprint.md`
- [ ] Meta title: 55-60 chars, keyword in first 40 chars
- [ ] Meta description: 150-160 chars
- [ ] H1 contains target keyword
- [ ] Target keyword appears in first 100 words
- [ ] Target keyword in 2-3 H2s (naturally)
- [ ] Pattern order followed (no missing required sections)
- [ ] 3+ data points with linked sources
- [ ] 3+ internal links to other Settley content
- [ ] 1-2 external links to FCA, GOV.UK, or authoritative sources
- [ ] Risk disclaimer present (required on all investment content)
- [ ] Waitlist CTA strip included
- [ ] CTA link and button copy updated for this article
- [ ] No return projections, no yield promises
- [ ] Word count meets target (±10%)
- [ ] Readability: no paragraphs longer than 4 lines
- [ ] Images: alt text includes keyword where natural
- [ ] FAQ schema kept only if FAQ answers are complete and accurate
- [ ] No duplicated title, category, or breadcrumb copy
- [ ] No generic stat row that does not change the reader's decision
- [ ] No more than 2 interactive embeds
- [ ] Primary interactive or visual proof block visible early on page
- [ ] Single CTA direction only

### Post-publish checklist

- [ ] URL submitted to Google Search Console for indexing
- [ ] Internal links added from 2 existing articles to the new one
- [ ] Twitter/X post scheduled
- [ ] Reddit share planned (if relevant thread exists — don't force it)

### Editorial cadence rule (new)

- Monday article = pillar / high-intent keyword
- Thursday article = supporting / trust / regulation / FAQ
- Every Thursday article must link back to Monday pillar

## 6. Metrics (Recalibrated)

### Realistic expectations for a new domain

Google applies a "trust period" to new domains — typically 3–6 months before meaningful ranking movement, regardless of content quality. The original targets were not wrong to aim for, but the internal narrative should be "foundation phase, not results phase" for months 1–3.

### Updated KPIs

| Metric | 30-Day Target | 60-Day Target | 90-Day Target | 6-Month Target |
|--------|-------------|-------------|-------------|--------------|
| Organic sessions/month | 50–150 | 200–600 | 500–1,500 | 3,000–8,000 |
| Keywords in top 20 | 3–5 | 8–15 | 15–25 | 40–60 |
| Keywords in top 10 | 0–1 | 2–5 | 5–12 | 20–35 |
| Keywords in top 3 | 0 | 0–1 | 1–3 | 5–12 |
| Published articles | 8 | 18 | 30 | 30+ |
| Referring domains | 3–8 | 10–20 | 20–35 | 40–60 |
| Waitlist signups (organic) | 5–15 | 20–40 | 40–80 | 150–300 |

> **What to report to Temisan:** Content volume, domain authority trend, keyword position movement. Not absolute traffic numbers in the near term.

### Tools stack (unchanged)
